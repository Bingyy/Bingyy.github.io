ADA提供的白皮书很奇怪，直接上来几篇学术论文，虽然很亲切....

从官网上，先看一些基础的东西。

Cardano是什么

Ada是代币的名称，实际归宿是Cardano。用途：发送、接收数字资金。

Cardano提供的是一个技术平台，能够运行金融程序，可以在上面构建DApp。

创新点是：架接用户和监管之间的平衡。

愿景：为所有人提供公平的金融服务。

Cardano的哲学

很少在区块链项目上看到这种赤裸裸的表述自己哲学的。

Cardano在2015年开始立项，目的是为了改变加密数字货币的设计与发展。

Cardano没有告知大家易于理解的发展路线，也没有权威的白皮书，但是有一些设计的原则，最佳的工程实践以及探索的渠道。

PS. 怎么很像没找程序员，就找了一堆哲学家画了饼的感觉？

主要包含以下几条：

  • 将会计和计算分为不同层次
  • 在高度模块化的功能代码中实现核心组件
  • 小组学者和开发人员与同行评审研究进行竞争
  • 大量使用跨学科团队，包括早期使用InfoSec专家
  • 迅速的迭代发生于白皮书，实施和需要通过审查期间发现问题进而更正的新颖研究
  • 建立具有在不破坏网络环境下进行升级部署后系统的能力
  • 制定未来运作所需的分散资金机制
  • 通过长期观点改进加密货币的设计，以便它们在移动设备上运行时，具有合理和安全的用户体验
  • 让利益相关者更加接近他们所拥有的加密货币之运营和维护
  • 承认需要对同一个分类帐中的多项资产负责
  • 交易包括可选元数据，以更佳符合传统系统的需求
  • 通过从将近1000种的山寨币中学习，含括其有意义的功能
  • 采用由互联网工程任务组启发的标准驱动流程，使用专门的基础来锁定最终的协议设计
  • 探索商业的社会元素
  • 为监管机构寻找一个健康的中间环节，与商业活动进行互动，并且不会影响从比特币继承的一些核心原则

本篇尝试不指出角色的名字的写法。

个人偏见来自于对整个剧情的设计厌恶。

但是不得不说女主的演技很好，完成度很高。

借用某豆瓣评论的一句话，看完以后整个感觉是这是一部大龄女青年色诱男孩的故事。

我知道很多人在歌颂这部片子里爱情的唯美，女主为了爱情的勇敢，以及男主的深情回馈。也有人将男主这种常人眼中的怪物物化为生活中具体的人，比如黑人，社会底层等等。甚至公主与马夫的爱情故事也可以套入这样的剧情架构中。

这样的爱情故事感人吗？

至少于我而言，喜欢不起来，甚至打心底里厌恶。

女主的这种行为，当然充满了圣母心。面对重重阻力，不畏强权，拯救心上人于危难之中，最后结局皆大欢喜，恶人得以惩戒，有情人终成眷属。完美极了。

但是看完后回头想一想，为了渲染爱情，导演用了多少女主裸体的场面。

从开始片子里不断渲染的女主早起必自慰一波的情节看，大龄女主在无限渴望着爱情的出现。这种场景，倒是道出了爱情里就是充满肉欲，如果喜欢，对彼此的身体自然充满性趣。这是写在基因里的本能，谁也无法回避。而不喜欢，往往从身体反应上就能看得出，体会得到。

女主这种就是寂寞难耐，充满欲望，渴望爱情的一面得到了强化，仿佛在告诉大家这不是一个清纯到白莲花一样的角色，也请不要用这样的期待来代入。

但是，后面剧情里女主强调，自己因为不能说话，生活中找不到懂她的人，实在不能理解。

论朋友，合租的画家，以及清洁工工友，都一直站在你身边，与你交心。你虽不能说话，却可以听，也可以通过手语交流，所以不过是不能通过声音来沟通而已。

好吧，你会说，这不是爱情式的沟通，也不是爱情式的懂。

于是你发现了一个常人眼中的怪物，但只有你慧眼识珠，发现了他的与众不同，爱上了他，他也爱上了你。很美的爱情桥段，但于我而言，不是很能接受这样的逻辑转变，幸好女主爱上的这个怪物最终值得她去爱。

也或许只是我对爱情的解构，越来越相信势均力敌，也越来越不相信爱情。

所以，我对此片的厌恶，无妨看作是对我自己的厌恶罢，至少女主还相信爱情；我却不敢再相信。

WSGI

发表于 2018-03-11 | 分类于 Flask

需要明白的是，Flask应用对象是WSGI应用程序的实例。

WSGI全称是Python Web Server Gateway Interface，是为Python语言定义的Web服务器与Web应用程序或框架之间的一种接口，简单且通用。

WSGI开发出来后，其他语言也出现了类似的接口。

WSGI出现以前，如何选择合适的Web应用程序框架是很让人困惑的，因为一旦选择了框架就会限制Web服务器的选择。

WSGI是作为Web服务器与Web应用程序或应用框架之间的一种低级别的接口。是基于现存的CGI标准而设计的。

WSGI规范一览

包含两个部分：

  • 服务器/网关
  • 应用程序/应用框架

既然是中间层，自然要对接这两个部分。

处理一个WSGI请求时，服务器会为应用程序提供环境信息以及一个回调函数。应用程序完成处理请求后，通过回调函数将结果返回给服务器。

从应用程序角度，WSGI扮演服务器，执行以下功能：

  • 重写环境变量，根据目标URL，将请求消息路由到不同应用对象
  • 允许在一个进程中同时运行多个应用程序or应用框架
  • 负载均衡和远程处理，通过网络转发请求和响应信息
  • 进行内容后处理
